# 载体平台选用微信小程序

MVP 以微信小程序为唯一载体，不做原生 App、不做独立 Web 站点。这个品类是「打开→转一下→走人」的工具型应用，安装摩擦致命，小程序的零安装决定能否用起来；v2 群体场景的杀手锏是「丢到群里一起选」，微信分享路径最短；CPS 变现也以微信生态内跳美团/饿了么小程序最稳。代价是受微信平台规则约束（佣金、合规、被封风险）、动画性能不如 native、重交互做不了——对该品类可接受。

## 考虑过的替代

- **Flutter/RN 双端原生**：动画与性能更好、独立于微信；但安装摩擦大、CPS 跨 App 跳转碎、双端开发量翻倍、上架审核。否决。
- **Web/PWA**：跨平台零安装；但微信内 Web 受限、定位与深链体验差、国内 PWA 生态弱。否决。
