# 技术栈选用 Taro

小程序用 Taro（React 语法）编写、编译为微信小程序，不用原生、不用 uni-app。驱动有二：一是团队 React 熟悉度——用 JSX/TS/React 生态写小程序，心智成本最低；二是多端期权——Taro 可编译到 App/支付宝/H5，为 v2/v3 可能的扩端留一条低成本切换路径（MVP 仍只发微信小程序，此为期权而非当前需求）。代价是 Taro 编译层有坑、微信新特性映射滞后、转盘等核心交互的性能调优要过编译层、大版本更迭有迁移成本——接受。

## 考虑过的替代

- **原生小程序**：零抽象、微信新特性即开即用、转盘性能调优最直接；但放弃 React 心智红利与多端期权。否决。
- **uni-app**：Vue 语法多端，能力与 Taro 同档；团队倾向 React 而非 Vue，否决。
