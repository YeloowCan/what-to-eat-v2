# 餐厅模型不含菜单、价格与下单

餐厅在本应用里只持有身份、位置与用户标注；菜单、价格与下单不在其内——它们是跳转到美团/饿了么后才发生的「平台侧事实」，不下沉到本地模型。之所以如此，是因为官方点单/菜单 API 对小团队不开放、POI 不提供菜单，而 CPS 联盟深链已能满足「选完即下单」；本地复制一份菜单既拿不到、也维护不动。

## 考虑过的替代

- **接入美团/饿了么官方菜单 API**：需企业资质 + 商务对接 + 佣金分成，对小团队不开放，否决。
- **爬取大众点评/美团菜单**：法律与稳定性双高风险，否决。
- **用户手填招牌菜/均价**：数据稀疏、维护成本高，且偏离「快速决策」的核心。若未来要做厚数据，可单开一条「用户标注」字段，但不进餐厅本体。
