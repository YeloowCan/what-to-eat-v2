# 今天吃什么 (What to Eat Today)

一个帮助用户从周边餐厅中快速做出「今天吃哪家」决策的 App。核心价值是消除选餐厅时的选择困难——给一个就够了。

## Language

**餐厅 (Restaurant)**:
一个供餐场所。在本应用的语境里，「餐厅」仅指场所的身份（高德 POI id）、位置和用户标注；菜单、价格与下单不在其内——它们是跳转到美团/饿了么后才发生的平台侧事实。照片、地址、营业时间等展示信息亦不在其内（见 [[展示信息]]）--按 poiId 现取、不持久化、不进决策快照。身份即 POI id，是去重与冷却所依据的稳定标识。
_Avoid_: 店铺、商家、餐馆、门店

**候选集 (Candidate Set)**:
一次决策中供随机抽取的一组餐厅，等于「周边 POI 检索结果」∩「用户当次约束」∩「冷却后剩余」。MVP 里从中均匀随机抽一家；v2 改为按偏好加权抽取。
_Avoid_: 选项列表、推荐池

**约束 (Constraint)**:
用户在一次决策中设置的筛选条件，用于把周边 POI 缩成候选集。MVP 维度为距离、营业中、菜系（皆 POI 出，零输入）；当次有效、不持久化。价格、口味等主观维度留待 v2，与用户标注一同引入。其中「营业中」与「菜系」为硬护栏——候选集为空时分层放宽只动冷却与距离，这两项永不自动放宽，要改须用户亲为。
_Avoid_: 筛选项、过滤条件

**决策 (Decision)**:
用户接受（点「就这家」）后落定的选择——在给定约束与冷却下，从候选集中选定一家餐厅。接受那一刻才写入本地（wx.storage），并快照该餐厅的身份（POI id）、位置、菜系与当次约束，使历史不依赖后续反查高德（店关改名仍可显示）；仅快照稳定事实，展示信息不入快照（见 [[展示信息]]）。决策既作冷却的依据，也作 v2 加权的输入；接受前转盘停下的结果只是「候选展示」，不算决策。
_Avoid_: 选择、抽签结果

**候选展示 (Suggestion)**:
转盘停下时展示给用户的那一家餐厅，是候选集的一次随机抽取结果。用户接受前它只是临时展示、不落库、不计冷却；被接受后才升格为「决策」。可被重抽覆盖。
_Avoid_: 结果、推荐

**冷却 (Cooldown)**:
近期已被决策选中的餐厅，会被临时排除出候选集，避免连续重复。由系统按本地决策历史自动计算，非用户设置——故与「约束」区分。
_Avoid_: 去重、屏蔽

**展示信息 (Display Info)**:
餐厅的照片、地址、营业时间、电话等展示用数据。不属于[[餐厅]]本体--与菜单/价格/下单同为「平台侧事实」，但属行动与可达性信息，按 poiId 在展示时现取、不持久化、不进[[决策]]快照（非稳定事实：图片 URL 会过期、营业时间会变）。取不到则降级（如菜系占位图）。人均/评分/招牌菜不在此列，留 v2 用户主观标签。
_Avoid_: 餐厅详情、富字段
