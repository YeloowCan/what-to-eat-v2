# 04 · 约束：距离 / 营业中 / 菜系（含硬护栏）

## Parent

- [PRD: 今天吃什么 MVP](../PRD.md)

## What to build

加**约束**选择器 UI：距离档（如 1/3/5km）、菜系枚举（火锅/川菜/快餐/日料/烧烤…，映射到高德 POI type）、营业中（默认开）。决策引擎按当次约束过滤候选集。约束当次有效、不持久化。营业中与菜系为**硬护栏**——后续分层放宽时这两项永不被自动改动，要改须用户亲为。用户也可不设任何约束直接转。

## Acceptance criteria

- [ ] 约束选择器可选距离档、菜系、营业中（默认开）
- [ ] 引擎按当次约束过滤候选集，转盘只给符合约束的餐厅
- [ ] 不设任何约束也能直接转盘
- [ ] 约束当次有效，下次打开默认重置（不持久化）
- [ ] 默认仅营业中餐厅出现在候选集
- [ ] 菜系枚举映射到高德 POI type code
- [ ] 引擎单测覆盖：约束过滤正确，且营业中/菜系被标记为硬护栏（护栏语义在本切片建立，分层放宽在 05 验证）
