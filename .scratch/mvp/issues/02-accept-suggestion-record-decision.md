# 02 · 接受候选展示，落定为决策并写入历史

## Parent

- [PRD: 今天吃什么 MVP](../PRD.md)

## What to build

在候选展示卡片上加「就这家」承诺点：用户接受 → 决策引擎生成一条**决策**记录（POI id + 餐厅名/坐标/菜系快照 + 接受时间戳 + 当次约束快照）→ 决策存储端口写入 `wx.storage`（滚动保留最近 50 条）→ 历史视图读取并展示最近选过的餐厅。重抽不落决策。匿名、无账号、无后端，状态只在本地。

遵循 ADR-0001（决策快照只含我们拥有的身份/位置/菜系，不含菜单/价格）。

## Acceptance criteria

- [ ] 候选展示卡片有「就这家」按钮；点击后该餐厅被记为一条决策
- [ ] 决策记录含 POI id、名/坐标/菜系快照、时间戳、当次约束快照；不含菜单/价格
- [ ] 决策写入 `wx.storage`，滚动保留最近 50 条（超出按时间淘汰）
- [ ] 历史视图能读取并展示最近选过的餐厅列表
- [ ] 即便某餐厅后来在高德关店/改名，历史记录仍显示当时快照的内容（不残缺）
- [ ] 重抽（不接受）不产生任何决策记录
- [ ] 全程匿名，无登录、无后端
