# 05 · 候选集为空的分层放宽

## Parent

- [PRD: 今天吃什么 MVP](../PRD.md)

## What to build

当「约束 ∩ 冷却」把候选集掏空时，决策引擎分层放宽并重抽：①先解冻冷却（系统自动态，最该先松）重抽 → ②仍空则逐步扩距离 1→3→5→10km 重抽 → ③仍空则向 UI 发「需放宽菜系」信号，UI 提示用户自行放宽。硬护栏营业中与菜系在任何层级都永不自动放宽——宁可无结果也不给打烊店、不偷换菜系。

## Acceptance criteria

- [ ] 候选集为空时，引擎先解冻冷却重抽
- [ ] 仍空则按 1→3→5→10km 逐档扩距离重抽
- [ ] 仍空则 UI 提示用户放宽菜系（交回用户，不自动改）
- [ ] 营业中在任何层级都不被自动放宽（永不给打烊店）
- [ ] 菜系在任何层级都不被自动放宽（用户明确指定的菜系不被偷换）
- [ ] 引擎单测覆盖：各层放宽触发顺序、硬护栏不被触碰、解冻/扩距离后能抽到结果
