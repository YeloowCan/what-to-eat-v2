# 01 · 转盘随机抽出一家周边餐厅

## Parent

- [PRD: 今天吃什么 MVP](../PRD.md)

## What to build

小程序首条端到端 tracer：打开 → 请求定位授权（拒绝则硬阻断全屏引导去系统设置）→ 决策引擎从周边 POI（无约束、默认半径 1km）均匀随机抽一家餐厅 → 转盘动画停下，展示**候选展示**（餐厅名、菜系、距离、营业状态）→ 用户可重抽（重新抽一家，不落任何记录）。

同时立起 Taro 工程骨架、决策引擎纯模块与单一测试 seam（POI 源 / RNG / 时间作为端口注入 fake）、位置端口（封装 `wx.getLocation`）、POI 源端口（封装高德周边 POI 搜索，安全 Key + appid 白名单，客户端直连）。POI 拉取失败或无网络时友好提示可重试。

遵循 ADR-0002（微信小程序）、ADR-0003（Taro）；餐厅模型不含菜单/价格（ADR-0001）。

## Acceptance criteria

- [ ] Taro 工程可在微信开发者工具运行，打开即进入定位授权流程
- [ ] 授权定位后，转盘能从周边真实 POI 中随机抽出一家并展示（名/菜系/距离/营业状态）
- [ ] 拒绝定位授权时，全屏硬阻断并引导去系统设置开启；不可绕过使用（无手选兜底）
- [ ] 重抽按钮可重新随机抽一家，且不写入任何决策记录
- [ ] 决策引擎为不碰 `wx.*` / 网络的纯模块，POI 源 / RNG / 时间以端口注入；有针对「从候选集均匀随机抽一家」的单测（种子 RNG 可断言抽中哪家）
- [ ] POI 拉取失败或无网络时显示友好提示并可重试，不白屏不崩溃
- [ ] 默认搜索半径 1km
